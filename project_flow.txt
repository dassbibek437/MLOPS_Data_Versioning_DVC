4. Now we do "dvc init" (creates .dvcignore, .dvc)
5. Now do "mkdir S3" (creates a new S3 directory)---->lets not make it compicated by adding aws directly ,just be simple for now 
6. Now we do "dvc remote add -d myremote S3"---if we are using aws then we have useed awsurl in the palce of S3
7. Next "dvc add data/" ---->Tell DVC to start tracking everything inside the data folder instead of Git.â€

What actually happens:

DVC calculates a hash (fingerprint) of all files inside data/.

The real data gets stored inside .dvc/cache/.

A metadata file called data.dvc is created.

The data/ folder becomes a tracked output (linked to cache).

.gitignore is automatically updated so Git stops tracking raw data files.
